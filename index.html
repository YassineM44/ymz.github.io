Overview

Medicore addresses a real clinical gap: affordable, portable, multi-metric monitoring without hospital infrastructure. The device simultaneously captures heart rate, SpO2, ECG, body temperature, and medication adherence in a single 3D-printed enclosure.

A dual-layer architecture cleanly separates real-time sensor polling on the ESP32 from data processing and display on the Raspberry Pi Zero 2W, eliminating memory contention and ensuring signal integrity under concurrent load.

Key Engineering Decisions

  • FreeRTOS task isolation — each of 5 sensors runs an independent async task on the ESP32, zero blocking delays
  • SPI/I2C serialization — structured binary payloads forwarded to the Pi gateway for processing, display, and logging
  • Autodesk Fusion enclosure — custom 15% infill, thermal escape channels, snap-fit PCB mounts built for field durability
  • Sensor fusion pipeline — MAX30102 + MLX90614 + AD8232 signals normalized and merged in Python for coherent output

Overview

Home Guard is a fully autonomous indoor safety robot built on STM32 — zero RTOS, zero HAL. Every peripheral is driven through direct hardware register manipulation (RCC, GPIO, USART, Timers), making the firmware deterministic and measurable down to the clock cycle.

The open-source peripheral driver library, released on GitHub as STM-Open-Library, provides the embedded community with modular, documented, register-level drivers for STM32 targets.

Key Engineering Decisions

  • Zero HAL / Zero RTOS — direct CMSIS register writes for GPIO, USART, I2C, and advanced Timer peripherals, full timing control
  • Non-blocking ISR design — sonar rangefinders and MQ gas sensors driven entirely by hardware interrupts, no polling loops
  • Tailscale VPN overlay — encrypted telemetry streamed to remote monitoring endpoints with zero public port exposure
  • Open-source library — structured, reusable peripheral drivers published on GitHub for the STM32 embedded community

Overview

Secure Fit is a smart bicycle lock prototype that bridges hardware relay control with a polished mobile-first interface. The system was designed UI-first in Figma — all user flows, auth states, and lock interactions validated before a single line of code was written.

A Raspberry Pi runs a Python state machine that evaluates authentication state and drives GPIO relay transitions to physically actuate the locking mechanism, with hard fail-safe defaults.

Key Engineering Decisions

  • UI-first methodology — complete Figma prototype covering onboarding, access token flow, and all lock states before implementation
  • State machine architecture — Python on RPi evaluates auth state and drives GPIO relay transitions deterministically
  • Fail-safe by default — relay coil defaults to energized (locked) on power anomaly or script failure, no unsafe open state possible
  • GPIO-to-relay interface — debounced GPIO signals driving relay coils with proper flyback protection on inductive loads
